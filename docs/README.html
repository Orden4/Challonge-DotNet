Challonge-DotNet documentation

The documentation is built using the docfx CLI. It automatically parses the xml documentation of the source code.

Extending the documentation

You can extend the documentation by writing markdown files, like described here.

IMPORTANT: The parsing engine used for this project is markdig, the default setting for docfx. It should not be changed.

Building the static html documentation

Follow the installation guide for docfx CLI. Once installed, from the project root, run

> docfx docfx/docfx.json

This will build the documentation and generate the static html pages in the docs folder.
